   6.01 — Typed directed edge schema. edge_types registry (16 standard
          types). provenance table with trust scoring. All Phase 2–4
          schema columns added now — one migration, ever. Two-pass FK-safe
          seeding. Historical backfill for existing records.

   6.02 — Personalized PageRank. fast-pagerank + scipy CSR. Hardware-
          tiered loading (Pi: 5K nodes, laptop: 50K, enterprise: 500K+).
          BFS fallback for graphs < 200 nodes. Alpha policy per pipeline
          step. PPR cache invalidation on every relate() write.

   6.03 — PPMI incremental counters. Lazy weight computation. Context
          smoothing α=0.75 (Levy et al. 2015). graph.prune() configurable
          via ONTO_PPMI_PRUNE_THRESHOLD env var (default 0.5).

   6.04 — Per-user decay profiles. decay_profiles table. Five seeded
          profiles: standard (λ=0.95), slow/HIPAA (λ=0.99), fast
          (λ=0.85), personal/GDPR (λ=0.97), financial/GLBA (λ=0.98).
          session_config override table with all future columns present.

   6.05 — Concept extraction evaluated. YAKE-inspired stdlib implementation
          selected. Zero new dependencies. Pluggable ConceptExtractor
          protocol. Phase 4 SpacyExtractor is a one-line swap.
